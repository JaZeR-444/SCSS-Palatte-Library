# Build Plan

Create a structured implementation plan BEFORE writing code.

Steps:
1. Restate the goal in one sentence.
2. Confirm it fits `rules/product-scope.md`.
3. List affected files/areas.
4. Identify protected decisions at risk (`references/protected-decisions.md`).
5. Outline the change in small, reviewable steps.
6. List tests to add/update.
7. Wait for approval before editing.

Return the plan only — do not edit yet.
